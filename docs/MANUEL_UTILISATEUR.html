1. Introduction et Connexion

1.1 Présentation

SGDA V5 est le système de gestion des aérodromes de l'ANACIM Sénégal. Il permet :

  • L'analyse du profil de risque de chaque aérodrome via 20+ modèles mathématiques et IA
  • La planification des surveillances (année en cours et N+1)
  • La gestion des écarts, PAC, certifications et homologations
  • Le kit documentaire réglementaire avec analyse IA
  • Le monitoring en temps réel des modèles d'apprentissage

1.2 Connexion

Deux modes d'authentification :

  • Email + mot de passe : identifiant au format prenom.nom@anacim.sn
  • Code d'accès : pour les exploitants et opérateurs temporaires
Après 5 tentatives échouées, le compte est temporairement bloqué. Contactez l'administrateur.

1.3 Navigation

La barre latérale gauche donne accès à tous les modules. Les modules visibles dépendent de votre rôle (voir section 9).

2. Profil de Risque

Module central du système. Le profil de risque est calculé automatiquement via 20+ modèles mathématiques et IA, mis à jour à chaque événement (écart, surveillance, incident).

2.1 Vue d'ensemble (tous les aérodromes)

  • KPIs globaux : score moyen des aérodromes, nombre d'aérodromes Excellents/Critiques/En dégradation
  • Analyse comparative : tableau de benchmarking classé par score, tri par C1-C5 ou région. Vue liste ou grille
  • Grille de cartes : chaque carte affiche le score, la tendance, le mini-radar C1-C5 (SGS, PAC, Conformité, Charge, Résilience), la maturité SGS (N1 Absent → N5 Efficace), les alertes HMM/Black Swan/Survival

2.2 Vue détaillée (clic sur une carte)

4 onglets pour l'analyse approfondie :

Onglet Synthèse

  • Gauge de risque : score global /100 avec niveau (Excellent ≥80, Bon 60-79, Modéré 30-59, Critique <30)
  • Radar C1-C5 : graphique radar des 5 critères
  • Tendance : amélioration (vert) ou dégradation (rouge)
  • Alertes IA : HMM (transition silencieuse), Survival (risque incident 90j), écarts critiques
  • Infrastructure : type d'entité, horaires, catégorie SSLIA
  • Tableau de synthèse multicritère : tableau C1-C5 avec poids, score, tendance par critère, prédictions 3m/6m avec intervalles de confiance, légende des niveaux

Onglet Diagnostic

  • C1-C5 détaillé : chaque critère avec barre de progression, score, explication du driver
  • Matrice OACI 5×5 : matrice de risque standard aviation, écarts positionnés dans les cellules avec compteurs
  • Corrélations C1-C5 : heatmap de corrélation entre critères, dépendance Copulas
  • Analyse Bow-Tie : pour chaque domaine — Dangers, Barrières préventives/correctives (avec efficacité %), Risque résiduel, Bénéfices estimés, Impact C1-C5
  • Black Swans : détection d'événements extrêmes improbables

Onglet Anticipation

  • Timeline des prédictions : scores projetés à 3, 6 et 12 mois
  • HMM : état caché (stable/dégradation/critique), risque de transition, jours avant criticité
  • Survival Analysis : hazard à 90 et 180 jours, médiane de survie
  • EVT (Extreme Value Theory) : risque extrême, distribution à queue lourde, maximum attendu 12 mois
  • Negative Binomial : surdispersion des incidents (arrivent-ils par grappes ?)
  • Prédictions contextuelles IA : probabilité par type d'incident (birdstrike, FOD, etc.) dérivée des données réelles
  • Simulateur de scénarios : sliders interactifs C1-C5, suggestions IA, sauvegarde de scénarios
  • Analyse des tendances : graphe d'évolution du score, R², points d'inflexion

Onglet Actions

  • Recommandations priorisées : basées sur le score global et chaque critère C1-C5
  • Modèles ML : Thompson Sampling (meilleure action), HMM (alerte transition), EVT (plan urgence), Survival (inspection 90j), Copulas (domaines liés), NB (grappes d'incidents)
  • Feedback inspecteur : évaluez la précision des prédictions (exacte/assez proche/très éloignée)
  • Exporter le rapport : téléchargement du rapport de risque au format texte
  • Recalculer le profil : force le recalcul immédiat du profil de risque (admin/inspecteur)

2.3 Vue DG (décideurs)

Pour les DG ANACIM et DG exploitants, une vue simplifiée est affichée :

  • Score global + niveau + tendance
  • Actions prioritaires (urgentes)
  • Points de vigilance
  • Projection du risque (3m, 6m, pire cas)
  • Écarts critiques

Les détails techniques (HMM, Copulas, Bow-Tie) sont masqués.

2.5 Analyse Bow-Tie — Données réelles vs structure générique

La carte « Analyse Bow-Tie — Barrières de défense » affiche une structure de barrières générique (2 préventives + 2 correctives par domaine), enrichie par des données réelles :

Ce qui est réel (data-driven)

  • Efficacité Maturité SGS : égale à C1 du profil de risque réel
  • Efficacité PAC : égale à C2 du profil de risque réel
  • Danger : basé sur le nombre réel d'écarts du domaine
  • Conséquence : basée sur C5 + les événements de sécurité réels du domaine
  • Probabilité résiduelle : formule combinant scoreGlobal et la moyenne C1/C2
  • Niveau de risque : selon échelle PAOE pour SGS, OACI pour les autres domaines

Ce qui est heuristique (structure fixe)

  • Nombre de barrières : toujours 2 préventives + 2 correctives, quels que soient le domaine et les données
  • Barrière « Audits {domaine} » : efficacité 70 si au moins une inspection existe, 30 sinon — règle arbitraire
  • Barrière « Nouvelles mesures (IA) » : efficacité = min(90, C2 + 15) — formule linéaire simple
  • Scénario : template « Si {défaillance} alors incident de sécurité {domaine} »
  • Gain estimé : (min(90, faible + 20) - faible) × 0.4 — heuristique, pas un calcul d'impact réel

Enrichissement bayésien (amélioration des efficacités)

Depuis la version 5.0, un réseau bayésien causal remplace les efficacités recopiées (C1/C2) par des probabilités causales inférées via élimination de variables :

  • Construction d'un réseau bayésien par bow-tie (nœuds = barrières + nœuds organisationnels)
  • Lissage Dirichlet (α=1.0) appliqué aux tables de probabilité conditionnelle dès la création
  • Variable Elimination : inférence exacte pour calculer P(Incident | État_barrières)
  • Nœuds organisationnels ancrés : charge_travail, formation_adequation, supervision_quality — visibles dans la section « Facteurs organisationnels » pour les domaines COP et SGS
  • Badge de confiance : chaque barrière affiche un indicateur de confiance du réseau (dots) sous son efficacité

Exclusion SGS non applicable

Si le statut SGS de l'aérodrome est « non applicable », le domaine SGS est absent du menu déroulant et de la vue globale. Le sélecteur n'affiche que les domaines disponibles pour l'aérodrome sélectionné.

2.6 Calcul du niveau de maturité SGS

Le niveau de maturité du Système de Gestion de la Sécurité (SGS) est évalué selon la méthode PAOE inspirée de l'Annexe 19 OACI :

Structure PAOE

Le SGS est décomposé en 5 composantes (piliers), chacune contenant des éléments, eux-mêmes composés de questions :

#ComposantePoids
1Politique et objectifs de sécurité20 %
2Gestion des risques de sécurité30 %
3Assurance de la sécurité25 %
4Promotion de la sécurité15 %
5Gestion des interfaces10 %

Échelle PAOE (5 niveaux)

NiveauScoreDescription
Absent0Le processus n'existe pas
Présent25Le processus existe mais est informel
Approprié50Le processus est documenté et appliqué
Opérationnel75Le processus est mesuré et maîtrisé
Efficace100Le processus est optimisé et en amélioration continue

Calcul par élément

Pour chaque élément d'une composante :

score_élément = moyenne(scores_des_questions) × (1 — hétérogénéité × 0,3)

où hétérogénéité = (max — min) / max des scores des questions de l'élément. Ce facteur pénalise la dispersion des réponses au sein d'un même élément.

Calcul par composante

score_composante = moyenne(scores_des_éléments) × (1 — hétérogénéité × 0,2)

Même principe : pénalisation de l'hétérogénéité entre les éléments (coefficient 0,2).

Score global de maturité SGS

score_SGS = Σ(score_composante × poids) / Σ(poids)

Moyenne pondérée des 5 composantes par leurs poids respectifs.

Correspondance score → niveau OACI

ScoreNiveau OACIAppellation
≥ 90N5Efficace
65 — 89N4Opérationnel
40 — 64N3Approprié
20 — 39N2Présent
< 20N1Absent
Exemple : Un aérodrome dont la composante « Gestion des risques » (poids 30 %) a un score de 72, « Assurance sécurité » (25 %) a 65, « Politique » (20 %) a 80, « Promotion » (15 %) a 50, « Interfaces » (10 %) a 45. Le score SGS global est : (72×0,30 + 65×0,25 + 80×0,20 + 50×0,15 + 45×0,10) / 1,00 = 65, soit N4 Opérationnel.

2.7 Calcul du profil de risque (C1-C5)

Le profil de risque combine 5 critères notés sur 100, chacun pondéré dans le score global :

CritèrePoidsDescription
C1 — Maturité SGS20 %Mesure la maturité du système de gestion de la sécurité
C2 — Efficacité PAC20 %Mesure la capacité à clôturer les écarts dans les délais
C3 — Conformité20 %Scores des dernières surveillances (la plus récente pèse 2×)
C4 — Charge d'écarts latents15 %Pondération des écarts actifs par niveau de criticité
C5 — Résilience25 %Historique des événements de sécurité

score_global = arrondi(C1 × 0,20 + C2 × 0,20 + C3 × 0,20 + C4 × 0,15 + C5 × 0,25)

C1 — Maturité SGS

Si la maturité est saisie sur une échelle de 1 à 5 : C1 = (maturité — 1) × 25

Si la maturité est déjà sur 100 : passage direct.

Si des enquêtes de perception existent : C1 = score_SGS × 0,7 + (score_enquête / 5 × 100) × 0,3

C2 — Efficacité PAC

Calculé à partir des écarts clôturés sur les 12 derniers mois :

C2 = moyenne(min(1, délai_échéance / délai_réel) × 100) pour chaque écart clôturé dans la période.

Si des scores d'évaluation PAC existent : C2 = C2_délai × 0,5 + PAC_moyen × 0,5

C3 — Conformité

Moyenne pondérée des 5 dernières surveillances. La plus récente a un poids double :

C3 = (score_récent × 2 + score_avant-dernier × 1 + …) / (somme_des_poids)

Par défaut (aucune surveillance) : 30/100.

C4 — Charge d'écarts latents

Niveau d'écartPoints de charge
Critique4
Élevé2
Moyen1
Faible0,5

charge = Σ(points_des_écarts_actifs)

C4 = max(0, 100 — min(100, charge / 50 × 100))

Si la charge dépasse 50, une décroissance logarithmique est appliquée pour éviter d'atteindre 0 trop rapidement.

C5 — Résilience

GravitéPoids
Accident / Critique40
Incident grave / Orange20
Incident / Jaune10
Panne / Gris5
Bleu3

C5 = max(0, 100 — Σ(poids × facteur_récence))

Facteur de récence : < 6 mois → 1,0 ; 6-12 mois → 0,75 ; > 12 mois → 0,50.

Niveau de risque

Score globalNiveauFréquence de surveillance
80 — 100Faible (Excellent)1× / an
60 — 79Moyen (Bon)2× / an
30 — 59Élevé (Modéré)4× / an
0 — 29Critique12× / an
Exemple concret — Aérodrome A :
• C1 = 65 (maturité SGS N4 Opérationnel)
• C2 = 72 (délais PAC respectés à 72 %)
• C3 = 80 (dernière surveillance : 85, avant-dernière : 70 → (85×2 + 70) / 3 = 80)
• C4 = 60 (2 écarts élevés × 2 = 4, 3 écarts moyens × 1 = 3, total 7 → 100 — 7/50×100 = 86, mais dégradé par l'âge de l'aérodrome → 60)
• C5 = 90 (1 incident grave il y a 8 mois : 20 × 0,75 = 15 → 100 — 15 = 85, légèrement rehaussé)

Score global = arrondi(65×0,20 + 72×0,20 + 80×0,20 + 60×0,15 + 90×0,25) = arrondi(13,0 + 14,4 + 16,0 + 9,0 + 22,5) = 75/100 → Niveau « Bon », surveillance 2×/an.
Exemple concret — Aérodrome B :
• C1 = 25 (maturité SGS N2 Présent, score (2-1)×25 = 25)
• C2 = 30 (écarts fréquemment hors délais, délai échéance/délai réel moyen = 0,30)
• C3 = 20 (aucune surveillance récente, valeur par défaut dégradée)
• C4 = 15 (1 écart critique × 4 + 3 élevés × 2 + 2 moyens × 1 = 12 → 100 — 12/50×100 = 76, mais dégradé par charges multiples et âge → 15)
• C5 = 10 (1 accident il y a 4 mois : 40 × 1,0 = 40 ; 2 incidents graves il y a 2 et 5 mois : 20×1,0 + 20×1,0 = 40 ; total 80 → 100 — 80 = 20, mais dégradé → 10)

Score global = arrondi(25×0,20 + 30×0,20 + 20×0,20 + 15×0,15 + 10×0,25) = arrondi(5,0 + 6,0 + 4,0 + 2,25 + 2,5) = 20/100 → Niveau « Critique », surveillance 12×/an.

3.2 Créer un planning

  1. Cliquer sur Nouveau planning
  2. Remplir le formulaire : aérodrome, type, dates, domaines, équipe
  3. Valider

3.3 Préparer une surveillance

Cliquer sur Préparer (icône fichier) sur une carte planning → ouvre la modale de préparation avec 4 onglets :

  • Profil de risque : score, tendance, critères C1-C5, vélocité
  • Historique : surveillances précédentes, écarts non résolus
  • Checklist : type recommandé (standard, SGS, suivi écarts, PAC), pré-remplissage IA
  • Délégation : assigner les domaines aux inspecteurs

Bouton Envoyer la checklist : notifie les exploitants par in-app + email.

3.4 Planning N+1 (année prochaine)

Disponible uniquement à partir de novembre de l'année en cours. Avant cette date, le bouton affiche « Bientôt ».

En novembre, le planning N+1 est généré automatiquement :

  1. Les propositions apparaissent classées par aérodrome (source : profil, carry-over écarts/PAC, certification)
  2. Valider : marque la proposition en vert (reste visible)
  3. Ajuster : modifier le type, les dates, les domaines, ajouter des observations
  4. Refuser : la proposition disparaît avec un motif
  5. Consolider le planning N+1 : bouton vert qui apparaît après validation → modale de confirmation avec toutes les propositions validées

3.5 Suggestions IA (année en cours)

Le bouton Suggestions IA (à droite de la barre d'outils) ouvre un panneau d'analyse pour l'année en cours :

  • Déclencheurs d'écarts par aérodrome (PAC, suivi, urgence)
  • Profil de risque avec type/fréquence suggérés
  • Bouton Créer maintenant pour les cas critiques
Le bouton pulse en rouge si des déclencheurs critiques sont détectés, en orange pour des alertes hautes, en bleu quand le panneau est ouvert.

5. Surveillance et Checklist

5.1 Lancement d'une surveillance

Depuis le planning, cliquer sur Exécuter (icône coche) → la checklist s'ouvre avec les items pré-remplis.

5.2 Types de checklist

  • Standard : items RAS-14 par domaine
  • SGS (PAOE) : évaluation de la maturité SGS selon l'Annexe 19 OACI (4 piliers, 5 niveaux)
  • Suivi des écarts : vérification de l'état d'avancement des écarts
  • Mise en œuvre PAC : vérification des plans d'action corrective
  • Maintien : items des surveillances antérieures + domaines du profil

5.3 Délégation des domaines

Avant de lancer la checklist, le chef d'équipe peut déléguer des domaines spécifiques à chaque inspecteur :

  1. Ouvrir la modale de préparation → onglet Délégation
  2. Pour chaque domaine, sélectionner l'inspecteur assigné
  3. Cliquer sur Enregistrer les délégations

Résumé des délégations visible dans la modale. Chaque inspecteur voit uniquement ses domaines assignés dans la checklist.

La délégation est optionnelle. Sans délégation, tous les inspecteurs peuvent voir l'ensemble des domaines.

5.4 Préparation de l'inspection — Côté Inspecteur

Depuis le module Planning, cliquer sur Préparer (icône fichier) sur une carte planning. La modale de préparation offre 4 onglets :

  • Profil de risque : score global, C1-C5, tendance, vélocité (vitesse, accélération, volatilité)
  • Historique : 3 dernières surveillances, écarts critiques non résolus
  • Checklist : type recommandé (standard, SGS PAOE, suivi écarts, PAC), pré-remplissage IA avec l'historique
  • Délégation : attribution des domaines aux inspecteurs

Ouverture de la checklist

Si plusieurs types de checklist sont disponibles, une modale de choix s'affiche :

  • Checklist Standard : items RAS-14
  • Évaluation SGS (PAOE) : maturité Annexe 19 OACI
  • Suivi des écarts : état d'avancement des écarts actifs
  • Mise en œuvre PAC : vérification des plans d'action corrective

5.5 Préparation de l'inspection — Côté Exploitant

L'exploitant reçoit une notification (in-app + email) quand une checklist est prête. Il peut :

  • Checklist de préparation (12 items interactifs) : vérifier les points avant inspection (documents, infrastructures, équipements, personnel)
  • Barre de progression : suivi en temps réel des items complétés
  • Badge « Prêt ! » : s'affiche quand tous les items sont validés
  • Persistance locale : les réponses sont sauvegardées dans le navigateur (localStorage)
L'exploitant reçoit des rappels automatiques à J-30, J-15 et J-7 avant la date de surveillance.

5.6 Feuille de présence et signature

Pendant l'inspection, la feuille de présence permet :

  • Signature électronique : pad de signature tactile (SignaturePadWithColor)
  • Enregistrement : date, heure, nom du signataire, rôle
  • Validation : le chef d'équipe valide la feuille signée
  • Archivage : la checklist signée est automatiquement archivée dans le registre

5.7 Rapports de surveillance

Après la complétion de la checklist et la signature :

  • Rapport automatique : généré avec tous les items, résultats (SA/NS/NV), écarts détectés
  • Export PDF : disponible dans le registre et dans la vue détail de la surveillance
  • Transmission : notification automatique à l'exploitant et au chef hiérarchique
  • Archivage : conservation dans le registre avec tous les fichiers attachés

7. ML Monitoring

Module réservé aux administrateurs pour le suivi des performances des modèles d'apprentissage.

7.1 Onglet Performances

  • KPIs : Feedbacks collectés, Précision du modèle, Alertes de recalibration, Échantillons RF, Convergence ML
  • Graphique : précision par domaine
  • Métriques : précision globale, faux positifs/négatifs, feedbacks récents
  • Corrélation ML ↔ Profil de Risque : convergence, alignement, distribution des niveaux
  • Panneau confiance réseau bayésien : état des nœuds bayésiens (lissage Dirichlet, inférence VE), nœuds COP avec scores de confiance
  • Alertes : recalibration en attente
  • Actions : Recalibrer, Réinitialiser

7.2 Onglet Modèles (admin only)

  • Random Forest : entraînement, précision, importance des caractéristiques, matrice de confusion. Si moins de 10 échantillons, le bouton Entraîner est désactivé avec un message « 0 échantillon RF — modèle non alimenté »
  • PAC Learning : taux de concordance, pondérations, priorisation
  • Badges source de données : chaque KPI affiche 🟢 Supabase ou 🟡 session selon l'origine
  • Historique : tableau des entraînements, graphique d'évolution de la précision, export CSV
  • A/B Testing : Neural Net vs Formules, taux de victoire, MAE
  • Configuration : entraînement automatique, intervalle, réinitialisation

9. Modèles Mathématiques, Agents IA et Architecture

9.1 Vue d'ensemble du pipeline IA

Tous les modèles mathématiques et agents IA du système convergent vers une source unique : le ProfilRisque. Le pipeline est le suivant :

  1. Données brutes : écarts, surveillances, événements, certifications, homologations, aérodromes
  2. Phase 1 — 13 modèles classiques : C1-C5 (score multicritère), régression linéaire, EWMA, Hawkes (contagion), CUSUM (changement brusque), Bayésien (probabilité postérieure), Poisson (nombre d'incidents), Corrélation, Scénarios (4 scenarios)
  3. Phase 2 — Métriques avancées : Vélocité (vitesse, accélération, volatilité), Stress système, Alertes proactives
  4. Phase 3 — 6 modèles ML (ModelCache partagé, TTL 5 min) : HMM, Survival, EVT, Negative Binomial, Copulas, Thompson Sampling
  5. Phase 4 — Ensemble IA (avec fallback automatique) : BayesianDynamic (≥5 points), XGBoost (≥20 points), LSTM (≥30 points)
  6. Bow-Tie : Barrières de défense par domaine

Résultat : ProfilRisque (30+ champs) → stocké en base → consommé par tous les modules.

10. Certification et Homologation

9.1 Certification

Gestion du cycle de vie complet des certifications d'aérodrome :

  • Demande initiale : soumission du dossier par l'exploitant
  • Phase 1 - Examen documentaire : vérification des documents réglementaires
  • Phase 2 - Inspection sur site : vérification physique des infrastructures
  • Phase 3 - Décision : certification, suspension ou refus
  • Renouvellement : suivi des dates d'expiration, alertes à 6/3/1 mois

Chaque phase génère des recommandations IA via le certificationAgent.

9.2 Homologation

Processus simplifié pour les aérodromes nationaux et hélistations :

  • Dossier d'homologation avec pièces justificatives
  • Évaluation technique par domaine
  • Émission du certificat d'homologation

9.3 Impact sur le profil de risque

Chaque mise à jour de certification ou d'homologation déclenche un recalcul automatique du profil de risque de l'aérodrome concerné.

12.2 Workflow Certification

  1. Demande → Phase 1 (documentaire) → Phase 2 (inspection) → Phase 3 (décision)
  2. Renouvellement : Phase 1 sautée (dossier existant), début direct à la Phase 2

12.3 Workflow Écart → PAC

  1. Détection d'un écart (checklist NS/NV)
  2. Ouverture de l'écart avec niveau de risque
  3. Proposition de Plan d'Action Corrective (PAC)
  4. Validation du PAC → Mise en œuvre → Soumission des preuves
  5. Évaluation des preuves → Clôture ou rejet
L'IA (riskAgent) assiste chaque étape : prédiction du résultat, suggestions de mesures, évaluation de l'efficacité.

12.4 Workflow Formation

  1. Détection de lacunes de compétences (via CompetenceMatrix)
  2. Suggestions de formation (via FormationSuggestions piloté par l'IA/profil de risque)
  3. Planification → Exécution → Évaluation
  4. Mise à jour automatique des compétences des inspecteurs

10. FAQ et Dépannage

Connexion

Q : « Identifiant ou mot de passe incorrect »
R : Vérifiez le format email (prenom.nom@anacim.sn). Si le problème persiste, contactez l'administrateur.

Q : La page ne charge pas / erreur 500
R : Si Supabase (free tier) est en pause, les données ne se chargent pas. L'administrateur doit réactiver le projet sur supabase.com.

Profil de Risque

Q : Pourquoi le score ne change pas ?
R : Le profil est recalculé automatiquement à chaque événement (écart, surveillance, incident). Utilisez le bouton « Recalculer » dans l'onglet Actions pour forcer une mise à jour.

Q : Que signifient les modèles HMM, EVT, etc. ?
R : Voir la légende dans l'onglet Synthèse (tableau de synthèse multicritère) et les explications dans chaque onglet. L'assistant IA peut aussi répondre à vos questions.

Planning N+1

Q : Le bouton N+1 affiche « Bientôt »
R : Le planning N+1 est disponible uniquement à partir de novembre de l'année en cours. Les propositions sont générées automatiquement.

Q : Comment valider définitivement les propositions N+1 ?
R : Validez chaque proposition (bouton vert), puis cliquez sur « Consolider le planning N+1 » → confirmez dans la modale.

Données

Q : Les données ne se chargent pas
R : Vérifiez votre connexion internet. Si le problème persiste, le projet Supabase est peut-être en pause (free tier).

Q : Puis-je exporter les données ?
R : Oui — bouton Exporter dans ML Monitoring, export CSV dans l'historique des entraînements, export rapport dans ActionsTab du profil de risque.

IA et Modèles

Q : Les prédictions sont-elles fiables ?
R : La précision augmente avec le nombre de données. Les modèles avancés (XGBoost, LSTM) s'activent automatiquement quand suffisamment de données sont disponibles (≥20-30 points). La régression linéaire classique prend le relais en attendant.

Q : Comment les modèles se coordonnent-ils ?
R : Tous les modèles convergent vers une source unique : le ProfilRisque. Un ModelCache partagé garantit qu'aucun modèle n'est calculé deux fois. Les résultats sont persistés en base de données.
